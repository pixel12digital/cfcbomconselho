# TESTES – PWA Aluno

## Objetivo

Este checklist valida as funcionalidades do **PWA do Aluno**, incluindo dashboard, visualização de aulas (teóricas e práticas), exames, financeiro e demais funcionalidades disponíveis para o aluno.

## Cenários Principais (rascunho)

- [ ] Cenário 1 – Login e acesso ao dashboard do aluno
- [ ] Cenário 2 – Visualização de aulas teóricas (turmas, presenças, frequência)
- [ ] Cenário 3 – Visualização de aulas práticas (agendadas e realizadas)
- [ ] Cenário 4 – Visualização de exames (status, resultados)
- [ ] Cenário 5 – Visualização de dados financeiros (faturas, parcelas, status)
- [ ] Cenário 6 – Responsividade mobile (layout em dispositivos móveis)

## Modelo de Caso de Teste

Para cada cenário, quando formos detalhar:

- **Cenário:** `<nome>`
- **Pré-condições:** `<o que precisa existir - ex.: aluno cadastrado, turma criada, etc.>`
- **Passos:**
  1. `<passo 1>`
  2. `<passo 2>`
  3. `<passo 3>`
- **Resultado esperado:** `<comportamento esperado após os passos>`
- **Status:** [ ] Pendente / [x] Aprovado / [ ] Falhou
