# TESTES – Módulo Financeiro (Admin)

## Objetivo

Este checklist valida as funcionalidades do **módulo Financeiro** no painel administrativo, incluindo criação de faturas, registro de pagamentos, controle de inadimplência, bloqueios automáticos e relatórios financeiros.

## Cenários Principais (rascunho)

- [ ] Cenário 1 – Criação de nova fatura (vinculada a aluno/matrícula)
- [ ] Cenário 2 – Edição de fatura existente
- [ ] Cenário 3 – Registro de pagamento (total ou parcial)
- [ ] Cenário 4 – Marcação automática de faturas vencidas (via job)
- [ ] Cenário 5 – Bloqueio automático de matrícula por inadimplência
- [ ] Cenário 6 – Bloqueio automático de agendamento de aula prática por inadimplência
- [ ] Cenário 7 – Visualização de resumo financeiro por aluno
- [ ] Cenário 8 – Relatórios financeiros (inadimplência, receitas)
- [ ] Cenário 9 – Cancelamento de fatura
- [ ] Cenário 10 – Consulta de histórico de pagamentos

## Modelo de Caso de Teste

Para cada cenário, quando formos detalhar:

- **Cenário:** `<nome>`
- **Pré-condições:** `<o que precisa existir - ex.: aluno cadastrado, matrícula criada, etc.>`
- **Passos:**
  1. `<passo 1>`
  2. `<passo 2>`
  3. `<passo 3>`
- **Resultado esperado:** `<comportamento esperado após os passos>`
- **Status:** [ ] Pendente / [x] Aprovado / [ ] Falhou
